# 쿠팡 파트너스 광고 배너 연동 가이드

> 다른 Next.js 프로젝트에서 재사용할 수 있도록 실제 구현 기준으로 작성.
> FerryCast 프로젝트에서 완성·검증된 코드 기준.

---

## 개요

쿠팡 파트너스 공식 G 위젯(`PartnersCoupang.G`)을 사용하는 캐러셀 배너.
배너 교체·인디케이터는 쿠팡 SDK가 자동 처리하므로 별도 구현 불필요.

---

## 쿠팡 파트너스 설정

1. [쿠팡 파트너스](https://partners.coupang.com) 로그인
2. 배너 생성 → **캐러셀(carousel)** 템플릿 선택
3. 생성된 스크립트에서 아래 값 복사:

```js
new PartnersCoupang.G({
  "id": 999288,           // ← 내 계정 ID
  "template": "carousel",
  "trackingCode": "AF7008655",  // ← 내 추적 코드
  "width": "320",
  "height": "100",
  "tsource": ""
})
```

## 구현 코드 (Next.js App Router)

### `components/CoupangSection.tsx`

```tsx
"use client"

import { useEffect, useRef } from "react"

export default function CoupangSection() {
  const containerRef = useRef<HTMLDivElement>(null)

  useEffect(() => {
    const container = containerRef.current
    if (!container) return
    container.innerHTML = ""

    const initWidget = () => {
      // g.js가 body에 직접 요소를 append하므로, 추가 전 목록을 기록해둠
      const before = new Set(Array.from(document.body.children))

      const observer = new MutationObserver(() => {
        for (const el of Array.from(document.body.children)) {
          if (!before.has(el) && el.tagName !== "SCRIPT") {
            observer.disconnect()
            container.appendChild(el) // 우리 카드 안으로 이동
            return
          }
        }
      })
      observer.observe(document.body, { childList: true })

      const s = document.createElement("script")
      s.textContent = `new PartnersCoupang.G({"id":999288,"template":"carousel","trackingCode":"AF7008655","width":"320","height":"100","tsource":""})`
      document.body.appendChild(s)

      return observer
    }

    let observer: MutationObserver | undefined
    const w = window as Window & { PartnersCoupang?: unknown }

    if (w.PartnersCoupang) {
      observer = initWidget()
    } else {
      const sdk = document.createElement("script")
      sdk.src = "https://ads-partners.coupang.com/g.js"
      sdk.async = true
      sdk.onload = () => {
        observer = initWidget()
      }
      document.head.appendChild(sdk)
    }

    return () => {
      observer?.disconnect()
      container.innerHTML = ""
    }
  }, [])

  return (
    <div className="rounded-2xl border border-slate-100 bg-white px-4 py-3 shadow-sm">
      <div ref={containerRef} className="min-h-[100px] w-full overflow-hidden" />
      <p className="mt-1 overflow-hidden text-ellipsis whitespace-nowrap text-[10px] leading-none text-slate-400">
        이 포스팅은 쿠팡 파트너스 활동의 일환으로, 이에 따른 일정액의 수수료를 제공받습니다.
      </p>
    </div>
  )
}
```

## 핵심 기술 노트

### MutationObserver 패턴이 필요한 이유

쿠팡 `g.js`는 `document.body`에 직접 위젯 DOM을 append한다.
React 컴포넌트의 `ref` 컨테이너에 넣으려면 DOM 변화를 감지해 이동시켜야 한다.

**흐름**:
1. `body`의 현재 자식 목록을 `before` Set으로 기록
2. `MutationObserver`로 `body` childList 감시 시작
3. G 위젯 초기화 스크립트를 `body`에 append
4. 새 요소(SCRIPT 제외)가 감지되면 → `containerRef`로 이동 → observer 해제

### g.js 중복 로드 방지

`window.PartnersCoupang` 존재 여부로 SDK가 이미 로드됐는지 확인.
이미 있으면 `g.js` 재로드 없이 바로 위젯 초기화.

### cleanup

컴포넌트 언마운트 시:
- `observer.disconnect()` — MutationObserver 해제
- `container.innerHTML = ""` — DOM 정리

## 다른 프로젝트 적용 시 교체할 값

| 항목 | FerryCast 값 | 교체 필요 |
|------|-------------|---------|
| `id` | `999288` | ✅ 내 파트너스 계정 ID로 |
| `trackingCode` | `"AF7008655"` | ✅ 내 추적 코드로 |
| `width` / `height` | `"320"` / `"100"` | 필요 시 조정 |
| `template` | `"carousel"` | 캐러셀 유지 권장 |
